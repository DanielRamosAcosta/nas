## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
`hardware-configuration.nix` is auto-generated by `nixos-generate-config`. Manually adding `fileSystems` entries there (like the cold-data subvolume mounts) risks being overwritten if the hardware config is ever regenerated.

Currently all cold-data subvolume mounts (immich, sftpgo, gitea, booklore, media, downloads, and soon postgres-backups) are defined in `hosts/nas/hardware-configuration.nix`.

These should be moved to a dedicated module (e.g. `hosts/nas/storage.nix` or `hosts/shared/storage.nix`) that is imported separately. NixOS merges `fileSystems` attrsets from all modules, so this works seamlessly.

**What stays in hardware-configuration.nix:**
- Root `/` mount (nixroot subvol @)
- `/home` mount (nixroot subvol @home)
- `/boot` mount (vfat)
- Swap, kernel modules, hardware-specific settings

**What moves to the new module:**
- All `/cold-data/*` subvolume mounts
